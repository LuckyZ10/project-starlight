# 什么是 Git

## 核心概念

Git 是一个**分布式版本控制系统**。它帮你记录文件的每一次修改，让你可以随时回退到之前的任何一个版本。

想象你在写论文：第一稿、第二稿、导师修改版、最终版……没有版本控制，你可能会有这样的文件：

```
论文.doc
论文_修改.doc
论文_最终.doc
论文_最终_真的最终.doc
论文_最终_真的最终_v2.doc
```

Git 就是来解决这个问题的。它帮你把每次修改都"存档"，而且不是简单复制文件，而是只记录**变化的部分**。

## 为什么选择 Git

1. **分布式**：每个人的电脑上都有完整的代码历史，不依赖中央服务器
2. **速度快**：大多数操作都在本地完成，不需要网络
3. **分支友好**：创建和切换分支非常轻量
4. **不可篡改**：每次提交都有唯一的哈希值（如 `a1b2c3d`），确保历史记录不被悄悄修改

## Git 的三个区域

理解 Git 最关键的概念是三个区域：

```
工作区 (Working Directory)    → 你正在编辑的文件
     ↓ git add
暂存区 (Staging Area)         → 下一次提交要包含的变更
     ↓ git commit
版本库 (Repository)           → 所有提交的历史记录
```

- **工作区**：你看到的文件夹，直接编辑文件的地方
- **暂存区**：你挑选好准备提交的变更（一个"购物车"）
- **版本库**：`.git` 目录里保存的所有历史提交

## 关键概念

- **仓库（Repository）**：一个项目的所有文件和历史记录
- **提交（Commit）**：一次"存档"，记录某一时刻所有文件的状态
- **分支（Branch）**：从主线分出来的独立开发线
- **远程（Remote）**：托管在服务器上的仓库副本（如 GitHub）

## 安装 Git

```bash
# macOS（使用 Homebrew）
brew install git

# Ubuntu / Debian
sudo apt install git

# 验证安装
git --version
```

## 常见误区

1. **Git ≠ GitHub**：Git 是版本控制工具，GitHub 是托管 Git 仓库的网站。你可以用 Git 不用 GitHub，就像你可以写邮件不一定要用 Gmail。
2. **Git 只能管理代码**：Git 可以管理任何文本文件（代码、配置、Markdown 文档等），但对二进制文件（图片、视频）的追踪效率较低。
3. **学会 Git 就不用备份了**：Git 管理的是版本历史，不是备份方案。推送到远程仓库可以起到一定备份作用，但不应替代真正的备份策略。
