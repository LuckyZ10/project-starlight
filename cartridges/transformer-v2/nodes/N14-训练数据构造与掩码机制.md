# 训练数据构造与掩码机制

> 本节你将学到：如何利用特殊标记定义序列边界，通过随机采样构建多样化的训练数据，以及如何使用注意力掩码确保模型计算的因果性与正确性。

## 核心概念

在训练语言模型时，我们需要将连续的文本语料转换为模型可以处理的批量数据。这不仅涉及到如何截取数据（数据构造），还涉及到如何在批量计算中处理长度不一且不能“窥视未来”的序列（掩码机制）。

## 详细说明

### 1. 序列边界标识：[BOS]与[EOS]
在处理文本序列时，模型需要明确知道一段文本从哪里开始，到哪里结束。`[BOS]`（Beginning of Sequence）和 `[EOS]`（End of Sequence）是特殊的起始和结束标记，专门用于标识序列的边界。

- **[BOS]**：通常添加在输入序列的最前方，作为模型触发生成的绝对起点。
- **[EOS]**：添加在序列的末尾，用于指示模型当前文本段的生成已经结束。

### 2. 训练数据的随机采样构造
在理论上，我们可以使用固定大小的“滑动窗口”在语料上逐字移动来获取训练数据。但在实际的工程实现中，为了提高数据效率，通常会从语料中**随机采样位置**作为窗口的起点，而不是严格地按顺序滑动。这种做法打破了固定文本块的局限，极大地增加了训练数据的多样性和随机性，有助于提升模型的泛化能力。

### 3. 注意力掩码与填充处理
在批量训练中，同一个 batch 内的序列长度往往不一致，需要通过填充（Padding）补齐到相同的长度；同时，自回归语言模型必须保证“因果性”，即当前时刻的预测只能依赖历史信息，不能“看到”未来的词。

为了同时解决这两个问题，工程上引入了**注意力掩码**机制：
- 在计算注意力分数时，通过掩码将**填充位置**（以及未来位置）的注意力分数设置为一个极大负值，即 $-\infty$。
- 经过 Softmax 函数计算后，这些极大负值对应的权重会趋近于零：$e^{-\infty} \approx 0$。
- 在具体的网络结构中（如 Transformer），这种掩码机制通常通过构建**下三角矩阵**来实现，既屏蔽了填充符的干扰，也确保了单向注意力计算的因果性和正确性。

## 数学推导 / 工程实现

注意力掩码的数学计算过程如下：

假设当前未掩码的注意力分数向量为 $S = [s_1, s_2, s_3, s_{pad}]$，其中 $s_{pad}$ 是填充位置的分数。

1. **应用掩码**：将填充位置的分数设为极大负值：
   $$S_{masked} = [s_1, s_2, s_3, -\infty]$$

2. **Softmax 计算权重**：
   $$Attention\_Weights = \text{Softmax}(S_{masked}) = \left[ \frac{e^{s_1}}{\sum e^{s_i}}, \frac{e^{s_2}}{\sum e^{s_i}}, \frac{e^{s_3}}{\sum e^{s_i}}, \frac{e^{-\infty}}{\sum e^{s_i}} \right]$$

3. **结果**：由于 $e^{-\infty} \approx 0$，填充位置的注意力权重最终变为 0，从而不会对有效的上下文信息计算产生干扰。

在代码实现中，PyTorch 等框架通常可以通过生成一个下三角矩阵（`torch.tril`）结合填充掩码来一键实现：

```python
import torch
import torch.nn.functional as F

# 假设 seq_len = 4，模拟未掩码的注意力分数（batch_size=1, num_heads=1, seq_len=4）
scores = torch.tensor([[[1.0, 2.0, 1.5, 3.0]]]) 

# 构造因果掩码（下三角矩阵为1，其余为0）
causal_mask = torch.tril(torch.ones(4, 4)).bool()
# 构造填充掩码（假设最后一个位置是填充符）
padding_mask = torch.tensor([[[True, True, True, False]]])

# 综合掩码：既要满足因果性，又要屏蔽填充位
combined_mask = causal_mask & padding_mask

# 将掩码为 False 的位置设为极大负值 (-1e9)
masked_scores = scores.masked_fill(~combined_mask, -1e9)

# 计算 Softmax
weights = F.softmax(masked_scores, dim=-1)

print("Masked Scores:\n", masked_scores)
print("Attention Weights:\n", weights)
# 结果中，未来位置和填充位置的权重均会趋近于0
```

## 常见误区

- **误区一：认为填充位置可以直接置零忽略**。在注意力机制中，如果直接将分数设为 0，经过 Softmax 的指数运算后，其权重依然会占据一部分比例（因为 $e^0 = 1$）。必须设为极大负值才能确保 Softmax 后趋近于 0。
- **误区二：混淆因果掩码与填充掩码的作用**。下三角矩阵（因果掩码）是为了保证模型不泄露未来信息；而将填充位设为极大负值（填充掩码）是为了排除无用字符的干扰。在工程实现中，两者通常需要通过逻辑与（AND）操作合并成一个综合掩码。

## 要点回顾
- `[BOS]` 和 `[EOS]` 是特殊的起始和结束标记，用于在数据处理时明确标识序列的边界。
- 相比于严格的滑动窗口，工程上常采用**随机采样位置**作为窗口起点的方式来构造训练数据，这能有效增加训练数据的多样性。
- 批量训练时通过**注意力掩码**将填充位置（及未来位置）的注意力分数设为极大负值，使其 Softmax 后权重趋近于零。这通常通过下三角矩阵等形式实现，以确保计算的因果性和正确性。
