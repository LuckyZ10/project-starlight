# 交叉熵损失与标签平滑

> 本节你将学到：交叉熵损失在 Transformer 中的梯度推导及其高效性，以及如何运用标签平滑技术防止模型过度自信。

## 核心概念：交叉熵损失的梯度特性

在 Transformer 模型的训练过程中，模型输出的原始分数称为 logits（记为 $z$），经过 Softmax 函数后转化为预测概率分布（记为 $q$）。真实标签通常表示为概率分布 $p$。

交叉熵损失对 logits 具有极为优美的梯度特性，其数学表达式为：

$$ \frac{\partial L}{\partial z_i} = q_i - p_i $$

**符号解释**：
- $\frac{\partial L}{\partial z_i}$：损失函数 $L$ 对第 $i$ 个 logit 值 $z_i$ 的梯度。
- $q_i$：模型在第 $i$ 个类别上输出的预测概率。
- $p_i$：真实标签在第 $i$ 个类别上的目标概率。

**计算高效性与并行能力**：
这个公式表明，任何一个输出位置的梯度仅仅等于**预测概率与真实标签之差**。这种极其简洁的代数形式使得梯度计算在数学推导和程序实现上都变得极为高效。此外，由于序列中各个输出位置的梯度计算相互独立，**所有位置的梯度可以并行计算**，这完美契合了 Transformer 架构高效并行训练的需求。

## 核心概念：标签平滑

在一般的分类任务中，真实标签通常采用 one-hot 编码（即正确类别为 1，其余类别为 0）。如果模型一味地去拟合这种极端分布，很容易导致模型变得“过度自信”，即对正确类别的预测概率无限逼近于 1，而对错误类别逼近 0。这往往会引发过拟合，导致泛化能力下降。

**标签平滑**是一种常用的正则化技术，用来缓解上述问题。它将单一的 one-hot 目标替换为一种由两部分组成的**混合分布**，其数学公式如下：

$$ p_i = \begin{cases} 1 - \varepsilon & \text{如果 } i \text{ 是正确类别} \\ \frac{\varepsilon}{V - 1} & \text{如果 } i \text{ 是错误类别} \end{cases} $$

**符号解释**：
- $\varepsilon$：平滑系数，是一个很小的超参数（典型取值为 $\varepsilon = 0.1$）。
- $V$：类别总数（词表大小）。

**示例**：假设一个 3 分类问题，真实类别是第 1 类。
- 原始 one-hot 分布为：$[1.0, 0.0, 0.0]$
- 应用标签平滑（$\varepsilon = 0.1$）后，目标分布变为：$[1 - 0.1, \frac{0.1}{2}, \frac{0.1}{2}] = [0.9, 0.05, 0.05]$

通过这种机制，模型被鼓励给错误类别保留一定的概率质量，从而有效防止模型过度自信，进而显著改善模型在未知数据上的泛化性能。

## 代码示例

下面通过一段完整的 Python 代码（基于 PyTorch），演示交叉熵损失的梯度计算以及标签平滑的实际应用效果：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

# 假设词表大小 V=3，当前正在计算 1 个样本的损失
V = 3
logits = torch.tensor([[1.5, 0.2, -0.5]], requires_grad=True)

# 1. 交叉熵损失的梯度特性演示
# 真实类别为 0（对应 one-hot: [1.0, 0.0, 0.0]）
target = torch.tensor([0])
loss = F.cross_entropy(logits, target)
loss.backward()

# 计算预测概率 q
probs = F.softmax(logits.detach(), dim=-1)
print("预测概率 q:", probs)
print("目标分布 p:", torch.tensor([[1.0, 0.0, 0.0]]))
print("实际梯度 ∂L/∂z:", logits.grad)
print("差值 q - p  :", probs - torch.tensor([[1.0, 0.0, 0.0]]))
# 可以观察到：实际梯度 == q - p

# 2. 标签平滑演示
epsilon = 0.1
smooth_target = torch.tensor([[1.0 - epsilon, epsilon / (V - 1), epsilon / (V - 1)]])
print("\n标签平滑后的目标分布 (epsilon=0.1):", smooth_target)
```

## 常见误区

- **混淆 Softmax 前后的变量**：在应用公式 $\frac{\partial L}{\partial z_i} = q_i - p_i$ 时，必须明确 $z_i$ 是 Softmax 之前的 **logits**，而 $q_i$ 是 Softmax 之后的**预测概率**。该梯度公式并不直接适用于 Softmax 之后的概率输出。
- **标签平滑的分布分配**：在使用标签平滑时，剩余的 $\varepsilon$ 概率质量是**平均分配**给除正确类别以外的所有 $V-1$ 个类别（即每个错误类别分得 $\frac{\varepsilon}{V-1}$），而不是随机分配或整体赋予某一个类别。

## 要点回顾

- 交叉熵损失对 logits 的梯度公式为 $\frac{\partial L}{\partial z_i} = q_i - p_i$（即预测概率与真实标签之差），计算极为高效。
- 在 Transformer 架构中，由于各位置独立，基于该公式可以做到所有位置的梯度并行计算。
- 标签平滑将 one-hot 目标分布替换为 $(1-\varepsilon)$ 和 $\frac{\varepsilon}{V-1}$ 构成的混合分布（典型值 $\varepsilon=0.1$）。
- 标签平滑的核心作用是防止模型过度自信，作为一种正则化手段改善模型的泛化性能。
