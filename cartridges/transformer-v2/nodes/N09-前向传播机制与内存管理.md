# 前向传播机制与内存管理

> 本节你将学到：Transformer编码器层前向传播的完整数据流与重复堆叠模式，以及深度网络训练中梯度检查点技术如何以计算换内存。

## Transformer中前向传播的完整数据流

在Transformer模型中，输入序列的数据流转路径如下：
1. **初始处理**：输入序列首先经过**嵌入层**和**位置编码**。
2. **层级处理**：处理后的数据随后进入 $N$ 个相同的编码器/解码器层。
3. **子层结构**：每层内部包含两个核心计算单元：**多头注意力子层**和**前馈网络子层**。
4. **连接与归一化**：为了保证梯度的稳定传播和特征融合，每个子层后接**残差连接**和**层归一化**。

## 编码器层前向传播的详细流程

在单个Transformer编码器层中，数据的具体处理步骤按照严格的顺序进行。其可视化流程如下：

- 输入 → 层归一化 → 多头注意力 → 残差连接 → 层归一化 → 前馈网络 → 残差连接 → 输出

**流程拆解**：
1. **输入**：接收上一层（或嵌入层）传来的张量。
2. **注意力阶段**：先进行**层归一化**，随后计算**多头注意力**，最后通过**残差连接**将该阶段的输出与原始输入相加。
3. **前馈阶段**：对上一步的结果再次进行**层归一化**，传入**前馈网络**进行非线性变换，最后同样通过**残差连接**加上前馈阶段的输入，得到最终的**输出**。

## “子层-归一化”的重复堆叠模式

Transformer之所以能够扩展到数十层甚至上百层的深度，得益于其优雅的架构设计。这种“子层-归一化”的重复模式构成了Transformer深度堆叠的基础。

模型通过将上述编码器层进行 $N$ 次串联堆叠，使得数据依次通过这 $N$ 个结构完全相同、但权重各自独立的层，从而不断提取和抽象更高维度的语义特征。

## 前向传播的内存管理

在实际工程中，前向传播的**内存管理**是一个巨大的挑战。

**为什么需要管理内存？**
在模型训练时，框架必须保存前向传播产生的**中间激活**，以供反向传播计算梯度使用。对于深度网络和大批量数据而言，这些激活消耗的内存可能远超模型参数本身所占用的内存。

**解决方案：梯度检查点技术**
为了解决内存不足的问题，工程上引入了**梯度检查点**技术。
- **核心思想**：以计算换内存。
- **工作机制**：在前向传播时，不再保存所有层的中间激活，而是**只保存关键层激活**。在反向传播需要用到未保存的激活时，再重新进行一次前向计算来恢复它们。这种方式大幅降低了训练时的显存峰值要求。

## 代码示例

以下是一个简化版的编码器层代码，展示了“输入 → 层归一化 → 多头注意力 → 残差连接 → 层归一化 → 前馈网络 → 残差连接 → 输出”的数据流过程：

```python
import torch
import torch.nn as nn

class EncoderLayer(nn.Module):
    def __init__(self, d_model, n_heads, d_ff):
        super().__init__()
        # 定义层归一化
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        
        # 定义多头注意力和前馈网络子层
        self.attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.ReLU(),
            nn.Linear(d_ff, d_model)
        )

    def forward(self, x):
        # 1. 注意力阶段：层归一化 -> 多头注意力 -> 残差连接
        x_norm1 = self.norm1(x)
        attn_output, _ = self.attn(x_norm1, x_norm1, x_norm1)
        x = x + attn_output  # 残差连接
        
        # 2. 前馈阶段：层归一化 -> 前馈网络 -> 残差连接
        x_norm2 = self.norm2(x)
        ffn_output = self.ffn(x_norm2)
        x = x + ffn_output   # 残差连接
        
        return x

# 假设参数: 维度512, 头数8, 前馈网络隐藏层维度2048
encoder_layer = EncoderLayer(d_model=512, n_heads=8, d_ff=2048)

# 模拟输入: batch_size=32, seq_len=10, d_model=512
dummy_input = torch.randn(32, 10, 512)
output = encoder_layer(dummy_input)
print(f"输出张量形状: {output.shape}") # 预期输出: torch.Size([32, 10, 512])
```

## 要点回顾

- Transformer前向传播的数据流依次为：输入经过嵌入层和位置编码，进入N个含有多头注意力和前馈网络的编码器/解码器层，且每个子层后接残差连接和层归一化。
- 单个编码器的标准流程为：输入 → 层归一化 → 多头注意力 → 残差连接 → 层归一化 → 前馈网络 → 残差连接 → 输出。
- 前向传播必须保存中间激活以供反向传播使用，激活内存可能远超参数内存。
- 梯度检查点技术通过只保存关键层激活并在反向传播时重计算其余部分，实现了以计算换内存的目的。
- “子层-归一化”的重复模式是Transformer能够实现深度堆叠的结构基础。
