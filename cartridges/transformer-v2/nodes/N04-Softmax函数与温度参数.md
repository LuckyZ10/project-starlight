# Softmax函数与温度参数

> 本节你将学到：Softmax函数的数学定义与三大性质，工程实现中解决溢出的数值稳定技巧，以及温度参数在文本生成中的作用。

## Softmax函数的定义与关键性质

Softmax函数将一组实数转换为概率分布。给定一个包含 $n$ 个元素的向量 $z = [z_1, z_2, ..., z_n]$，Softmax的计算公式为：

$$ \text{Softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^n e^{z_j}} $$

其中 $z_i$ 是输入向量的第 $i$ 个元素，$e$ 是自然常数，分母是所有元素指数的和。

该函数具有三个关键性质：
1. **输出恒为正**：由于指数函数 $e^x$ 的值域始终大于0，Softmax的每一项输出都严格大于0。
2. **输出之和为1（归一化）**：分母是所有分子之和，因此所有输出元素相加等于1，使其结果可以直接解释为概率分布。
3. **保持相对顺序（单调性）**：指数函数是单调递增的，输入元素之间的大小关系在变换后保持不变。即原来较大的输入，经过Softmax计算后输出的概率依然较大。

## 数值稳定性问题及解决方案

在工程实现中，**数值稳定性**是一个必须面对的关键挑战。

当输入值 $z_i$ 较大时，计算 $e^{z_i}$ 极易导致浮点数溢出（变为 `inf` 或 `NaN`）。为了解决这个问题，标准的解决方案是采用**“减去最大值”技巧**。

具体做法是在计算指数之前，将输入向量中的每一个元素都减去该向量中的最大值 $z_{max}$。修正后的计算公式为：

$$ \text{Softmax}(z_i) = \frac{e^{z_i - z_{max}}}{\sum_{j=1}^n e^{z_j - z_{max}}} $$

这一变换在数学上不改变最终的结果（分子分母同时乘以了 $e^{-z_{max}}$），但它将所有的指数输入都压缩到了非正区间（即 $z_i - z_{max} \le 0$），使得 $e^{z_i - z_{max}}$ 的值落在 $(0, 1]$ 之间，从而彻底消除了浮点数溢出的风险。

## Softmax温度参数的应用

在文本生成任务中，通常会引入温度参数 $T$ 来控制Softmax输出分布的“平滑度”或“尖锐度”，计算公式拓展为：

$$ \text{Softmax}(z_i, T) = \frac{e^{z_i / T}}{\sum_{j=1}^n e^{z_j / T}} $$

其中 $T$ 控制了生成文本时的随机性与确定性，具体应用场景如下：
*   **T = 0.7 - 1.0**：用于平衡生成文本的**质量与创造性**，使模型在保持语句通顺的同时，具备一定的词汇多样性。
*   **T = 0.2 - 0.5**：用于**事实性任务**（如代码生成、知识问答、数据提取）。较低的温度使概率分布更集中，模型倾向于选择概率最高的词，从而保证输出结果的准确性和确定性。

## 代码示例

以下代码演示了Softmax的基本实现、减去最大值技巧防止溢出，以及温度参数对输出分布的影响：

```python
import numpy as np

def softmax(logits, temperature=1.0):
    # 应用温度参数
    scaled_logits = logits / temperature
    
    # 减去最大值技巧：解决数值稳定性问题，防止溢出
    max_logit = np.max(scaled_logits)
    stable_logits = scaled_logits - max_logit
    
    # 计算指数
    exp_logits = np.exp(stable_logits)
    
    # 计算概率分布 (输出恒为正，且之和为1)
    probabilities = exp_logits / np.sum(exp_logits)
    return probabilities

# 测试数据
z = np.array([2.0, 1.0, 0.1])

print("标准温度 (T=1.0):")
print(softmax(z, temperature=1.0))
# 输出性质验证: 元素均大于0，总和为1，且大小顺序与原数组一致

print("\n低温 - 事实性任务 (T=0.5):")
print(softmax(z, temperature=0.5))
# 输出分布更尖锐，最大值概率被进一步拉高

print("\n存在极大值时的数值稳定性测试:")
z_large = np.array([1000.0, 1001.0, 1002.0])
print(softmax(z_large, temperature=1.0))
# 若不减去最大值，这里会报错溢出 (RuntimeWarning: overflow encountered in exp)
```

## 常见误区

*   **忽略数值溢出问题**：在编写深度学习底层算子时，如果直接代入原公式计算 $e^{z_i}$，一旦输入特征值较大（如超过700），程序就会因为浮点数上溢而崩溃。
*   **误认为减去最大值会改变概率结果**：数学推导可以证明，$e^{z_i - z_{max}}$ 的比例关系与 $e^{z_i}$ 完全一致，该技巧仅仅是为了让计算机能安全地算出结果。
*   **混淆温度参数的作用方向**：温度 $T$ 越小，分布越“尖锐”（倾向于确定性输出）；$T$ 越大，分布越“平滑”（倾向于随机性输出）。

## 要点回顾

*   **三大性质**：Softmax输出恒为正、所有输出元素之和为1（归一化）、输入输出的相对大小顺序保持不变（单调性）。
*   **数值稳定性**：通过“减去最大值”技巧，将指数的输入压缩到非正区间，可以在不改变数学结果的前提下彻底消除浮点数溢出的风险。
*   **温度参数应用**：在文本生成中，$T=0.7-1.0$ 用于平衡生成内容的质量与创造性；$T=0.2-0.5$ 用于事实性任务以确保结果的准确性。
