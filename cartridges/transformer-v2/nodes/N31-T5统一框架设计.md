# T5统一框架设计

> 本节你将学到：T5（Text-to-Text）框架如何通过添加特定前缀，将分类等不同NLP任务统一转换为纯文本到文本的生成格式。

## 核心概念

T5（Text-to-Text Transfer Transformer）模型的核心设计理念是“**文本到文本**（Text-to-Text）”。在传统的深度学习中，不同的自然语言处理任务（如分类、翻译、摘要等）通常需要设计不同的模型输出层和目标格式。

T5框架通过一种极简的工程手段打破了这种边界：**将所有任务的输入和输出都统一为纯文本字符串**。这样一来，无论是分类、问答还是翻译，模型始终只需执行同一个基础任务——根据输入的文本字符串，生成输出的文本字符串。

## 详细说明

在T5框架下，为了让模型区分当前需要处理的具体任务，必须在输入文本前添加**特定的任务前缀**。

以**分类任务**为例，模型本身并不内置“分类”这一概念，而是通过前缀指示模型执行相应的操作：
- **输入格式**：`"sentiment: " + 句子`
- **输出格式**：`"positive" / "negative"`

通过在原始句子最前方拼接 `"sentiment: "` 前缀，模型能够识别出当前任务是一个情感分类任务。相应地，模型也不再输出概率分布或者特定的类别ID，而是直接生成代表情感极性的自然语言单词，例如 `"positive"` 或 `"negative"`。

通过这种“加前缀 -> 文本生成”的范式，T5极大地简化了模型架构，实现了用一个统一的生成式模型同时应对所有NLP任务的工程目标。

## 代码示例

以下是使用T5框架处理分类任务时的输入输出数据构造示例：

```python
# 原始数据
original_sentence = "This movie is absolutely wonderful and inspiring."

# 构造 T5 输入格式：添加 "sentiment: " 前缀
input_text = f"sentiment: {original_sentence}"

# 模型生成的目标输出（纯文本）
target_output = "positive"

# 另一个例子
negative_sentence = "The food was terrible and the service was bad."
input_text_2 = f"sentiment: {negative_sentence}"
target_output_2 = "negative"

print("模型输入:", input_text)
print("期望输出:", target_output)
print("模型输入:", input_text_2)
print("期望输出:", target_output_2)
```

## 常见误区

- **混淆模型输出类型**：传统分类模型通常输出对应类别的索引（如 `0` 或 `1`）或通过Softmax输出的概率值；但在T5框架中，输出必须被转化为具体的自然语言文本（如字符串 `"positive"` 或 `"negative"`），模型执行的是生成任务。
- **忽略前缀格式**：输入前缀是引导T5模型区分不同任务的关键标识。如果直接输入句子而不添加 `"sentiment: "` 前缀，模型将无法正确理解任务上下文，从而输出不可控的随机文本。

## 要点回顾

- T5（文本到文本框架）将所有NLP任务统一为“输入文本 -> 生成输出文本”的单一格式。
- 处理分类任务时，输入格式必须为添加 `"sentiment: "` 前缀的句子。
- T5模型直接输出生成的具体文本标签，例如 `"positive"` 或 `"negative"`。
