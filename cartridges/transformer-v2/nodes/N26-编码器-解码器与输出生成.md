# 编码器-解码器与输出生成

> 本节你将学到：交叉注意力机制中权重矩阵的形状与词对齐意义，以及输出层中权重绑定策略的实现与优势。

## 核心概念一：交叉注意力与词对齐

在编码器-解码器架构中，解码器需要根据编码器生成的源序列表示来生成目标序列。这一过程通过**交叉注意力**机制实现。

交叉注意力在计算时，Query ($Q$) 来自解码器（目标序列），而 Key ($K$) 和 Value ($V$) 来自编码器（源序列）。计算得到的注意力权重矩阵形状为 $[target\_len, source\_len]$：
- $target\_len$：目标序列的长度（矩阵的行数）。
- $source\_len$：源序列的长度（矩阵的列数）。

**词对齐意义**：该矩阵中的每一个元素 $\alpha_{i,j}$ 代表在生成目标序列的第 $i$ 个词时，对源序列的第 $j$ 个词分配的注意力比重。在对权重矩阵进行可视化时，能够清晰地反映出源词与目标词之间的对齐关系。例如在机器翻译任务中，通过该矩阵可以直观地看到目标语言生成的某个词究竟“关注”了源语言句子中的哪几个词。

## 核心概念二：输出层权重绑定

在模型最后的输出层，通常需要使用一个大型权重矩阵 $W_{vocab}$ 将隐藏状态映射到整个词表的空间以计算各个词的概率。

**权重绑定策略**：为了让模型更高效，可以让 $W_{vocab}$ 与输入嵌入矩阵共享权重（或转置共享）。
- **输入嵌入矩阵**：形状通常为 $[vocab\_size, d\_model]$，负责将离散的词 ID 映射为稠密的词向量。
- **输出权重矩阵**：形状通常为 $[d\_model, vocab\_size]$，负责将隐层状态映射回词表大小的概率分布。
- **转置共享**：即令 $W_{vocab} = E_{embed}^T$，两者在内存中共享同一套参数。

## 收益分析

输出层权重绑定策略能带来两个核心收益：
1. **减少参数量**：词表通常很大（如几万到几十万），嵌入层和输出层各自包含海量参数。通过共享矩阵，可以直接节省数千万甚至上亿的模型参数，显著降低显存占用。
2. **促进语义空间一致性**：在未绑定的模型中，输入层的词向量与输出层的预测矩阵往往处于不同的语义空间。强制共享权重使得“输入侧理解词”和“输出侧预测词”在同一向量空间中进行，确保了语义表达的一致性，有助于模型性能的提升。

## 常见误区

- **混淆注意力计算方向**：在交叉注意力中，权重矩阵的行数是目标序列长度 $target\_len$，列数是源序列长度 $source\_len$，不可颠倒。
- **仅关注减少参数量**：权重绑定的目的不仅是节省内存，促进输入与输出的语义空间一致性同样是该策略的关键理论收益。

## 要点回顾
- 交叉注意力权重矩阵的形状为 $[target\_len, source\_len]$，其元素值可视化后能清晰反映源语言与目标语言的词对齐关系。
- 输出层的权重绑定策略是指 $W_{vocab}$ 与输入嵌入矩阵共享（或转置共享）。
- 权重绑定的两大收益为：大幅减少模型的总参数量，并促进输入与输出阶段的语义空间一致性。
