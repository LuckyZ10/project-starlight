## 阶段规划

### Phase 0: 设计规划（当前）
- [x] 项目愿景和核心决策确认
- [x] 架构方案确认（插件化微内核）
- [ ] **T1: Harness 深度设计** — 参考 24 个启智 skill，输出完整设计文档
- [ ] **T2: 卡带协议细化** — manifest.json 的完整 schema、校验规则
- [ ] **T3: 数据库设计** — 用户、进度、卡带、贡献者、计费的表结构
- [ ] **T4: API 设计** — Adapter 接口、Web API 路由

### Phase 1: 核心引擎
- Harness 实现
- 卡带引擎实现
- DAG 调度
- 进度持久化

### Phase 2: Telegram 入口
- Telegram Bot Adapter
- 基础指令集（/start, /study, /progress）
- 对话式考核完整闭环

### Phase 3: 示例卡带
- 1-2 个完整卡带（Python 基础？经济学？）
- 通关测试
- 贡献者致敬

### Phase 4: Web 平台
- REST API
- 卡带编辑器（Web）
- 用户仪表盘（进度、证书）

### Phase 5: 社区 & 计费
- PR 式审核流程
- 社区投票
- 套餐 + token 包计费
- 用户自带 API key

### Phase 6: 扩展
- 微信支持
- 卡带制造 Agent
- 更多考核模式（代码执行、选择题）
- 排行榜、成就系统

## 原则

- **设计先行**：所有模块先出设计文档，确认后再写代码
- **质量第一**：参考启智研究成果，不抄但要借鉴
- **一起行动**：规划好后一起推进，不零散开发
- **可测试**：每个模块都要有测试用例
